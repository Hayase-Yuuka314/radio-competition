# DECISIONS — 架构决策记录

## ADR-001：选择单载波 BPSK 作为初始基线

- **状态**: accepted
- **背景**: 需要快速建立可测量基线，确保后续 ML 增强有可靠的 fallback。
- **候选方案**: 单载波 BPSK、单载波 QPSK、OFDM。
- **评估场景**: 理想信道、AWGN、CFO。
- **结果**: BPSK 实现最快、同步最简单、调试最直观。
- **决策**: 第一版使用单载波 BPSK，稳定后升级为 QPSK，OFDM 作为后期可选分支。
- **代价**: 峰值吞吐率较低，但不影响先建立稳定基线。
- **回退条件**: 不适用（这是起点）。

## ADR-002：使用卷积码 (rate 1/2, K=7) 作为首版 FEC

- **状态**: accepted
- **背景**: 需要一种比重复码更有效的 FEC，同时实现复杂度可控。
- **候选方案**: 无编码、重复码、卷积码、LDPC。
- **评估场景**: AWGN 信道。
- **结果**: 卷积码实现相对直接，维特比译码器 CPU 友好。
- **决策**: 首版使用 NASA-DSN 标准卷积码。
- **代价**: LDPC 可能提供更好性能，但开发成本高，作为后期比较。
- **回退条件**: 若实时性不足，降为更高码率或减少迭代。

## ADR-003：Python 3.11 + NumPy/SciPy 作为核心工具链

- **状态**: accepted
- **背景**: 需要在快速开发和运行效率间取得平衡。
- **候选方案**: Python、C++/GNU Radio、MATLAB。
- **决策**: Python 3.11 + NumPy 向量化可满足实时处理需求，CPU 优先不考虑 GPU。
- **代价**: 纯 Python 某些 DSP 操作较慢，关键路径可能需要 C 扩展或 Numba。
- **回退条件**: 实测实时因子 > 1 时考虑 C 扩展。

## ADR-004：ML 模型必须有无模型回退路径

- **状态**: accepted
- **背景**: 计划文档明确要求 ML 失败时自动回退。
- **决策**: 所有 ML 输出都必须经过置信度门控、OOD 检测和滞回，失败时回到 P1 Robust Profile。
- **代价**: 增加了系统复杂度，但保证了比赛可靠性。
- **回退条件**: 不适用（这是约束而非选择）。
